## 🎯 Сценарии использования

### Сценарий 1: "Нужно быстро проверить PR"
1. Открой **[quick-reference.md](./quick-reference.md)**
2. Используй чеклист на 5-15 минут
3. Выполни команды поиска проблем

### Сценарий 2: "Нашел уязвимость, нужны детали"
1. Открой **[security-vulnerabilities.md](./security-vulnerabilities.md)**
2. Найди нужный тип уязвимости
3. Посмотри примеры и решения

### Сценарий 3: "Код медленный, где искать проблему"
1. Открой **[performance-issues.md](./performance-issues.md)**
2. Проверь раздел по категории (аллокации/алгоритмы/копирование)
3. Используй инструменты профилирования

### Сценарий 4: "Нужно улучшить архитектуру"
1. Открой **[code-quality.md](./code-quality.md)**
2. Раздел "Архитектурные принципы"
3. Проверь SRP, модульность, DI

### Сценарий 5: "Rust-специфичная проблема"
1. Открой **[rust-specific.md](./rust-specific.md)**
2. Найди категорию (ownership/lifetimes/unsafe)
3. Посмотри примеры правильного кода

### Сценарий 6: "Хочу изучить на примерах"
1. Открой **[examples.md](./examples.md)**
2. Читай пример Telegram Authentication
3. Смотри метрики улучшений

## 🔄 Процесс code review

```
┌─────────────────────────────────────────────┐
│  1. Открыть PR и прочитать описание         │
│     Время: 2-3 минуты                       │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  2. БЕЗОПАСНОСТЬ (security-vulnerabilities.md) │
│     - Секреты, инъекции, валидация         │
│     Время: 5-10 минут                       │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  3. ПРОИЗВОДИТЕЛЬНОСТЬ (performance-issues.md) │
│     - Дублирование, O(n²), аллокации       │
│     Время: 5-10 минут                       │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  4. КАЧЕСТВО (code-quality.md)              │
│     - DRY, читаемость, тесты               │
│     Время: 5-10 минут                       │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  5. RUST-СПЕЦИФИКА (rust-specific.md)       │
│     - Ownership, unsafe, lifetimes         │
│     Время: 5 минут                          │
└──────────────────┬──────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│  6. Написать комментарии                    │
│     - Конкретно, с примерами решений       │

                   │
```

## 📖 Как читать

### Новичок в code review:
1. README.md - поймешь общий подход
2. quick-reference.md - запомни основные проверки
3. examples.md - изучи на примерах
4. Остальное - по мере необходимости

### Опытный reviewer:
1. quick-reference.md - используй как checklist
2. Специфичные разделы - когда нужны детали
3. examples.md - для референса метрик
